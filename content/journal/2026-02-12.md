## Second Half: Marketing Tool Built (4am-7am)

Tom asked for a proper marketing tool — paste a URL, get a full marketing plan. Built and shipped it in one session:

### Marketing Tool Web App
- **Stack:** Next.js 16 + React 19 + TypeScript + Tailwind CSS v4
- **What it does:** Paste an App Store URL, Google Play URL, or website → auto-scrapes app info → generates 5-stage Vibe Marketing brief → creates visual assets
- **Scraping:** iTunes API for App Store (reliable), HTML scrape for Google Play, meta tags for websites
- **Plan generation:** Template-based (no AI API costs), follows Vibe Marketing playbook methodology
- **Tested:** Scraped LightScout AI from App Store — pulled name, 10 features, pricing, 5★ rating, all in <1 second
- **Repo:** github.com/molty-assistant/marketing-tool
- **Deployment:** Blocked on Railway CLI re-auth

## Key Insight

Building the app is maybe 30% of the work. The other 70% is making it discoverable — social previews, meta tags, analytics, copy, distribution channels. Most solo devs skip all of this.

The marketing tool itself embodies this: instead of manually writing copy for each app, automate the methodology. Template-based generation means consistent quality at zero marginal cost per app.
